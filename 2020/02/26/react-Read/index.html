react 高阶组件

高阶组件在于将逻辑与UI分离，从而使得UI的复用．高阶组件其实就是一个纯函数，将组件作为参数传递后返回一个新的组件．
高阶函数有两种的实现方式：

  • 属性代理
    　属性代理可以返回一个无状态组件或者一个class组件
  • 反向继承
    反向继承通过继承原有的组件，可以获得其相应的状态与生命周期
    因为在js中class的继承都只是语法糖，其实实现的方式还是通过原型链来实现的，所以在继承中可以获取到原组件的相应的方法，进而通过只改变自己的数据改变UI的渲染
    通过super.render()可以获取原组件的render方法

React提升应用性能

  • 一致性比较与key属性

    react调用自身的渲染方法，并且会递归调用组件的渲染方法．组件的渲染方法会返回React元素树，然后React根据此来判断更新UI需要执行那些 DOM操作

    当组件的状态发生变化时，会再次调用组件的渲染方法，并与之前的React元素树进行比较．能计算出产生预期最小的变化的操作集合，这个过程称为一致性比较．

    在下面的情况中，click事件触发后，在state后面加上一个baz,而后面的一个将在数组的前面加上baz．

    前者并没有引起页面的重新渲染，而后者进行了重新渲染．React在比较的过程中，它只是检查前后两次的子元素是否相同，若发现第一个元素不同，他就会认为所有的列表项都是新的，因此全部修改．
